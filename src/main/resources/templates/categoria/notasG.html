Documentación Breve de Archivos HTML en la Carpeta /categoria

1. crear.html

Descripción: Este archivo permite al usuario registrar una nueva categoría.

Componentes:

  • Formulario: Contiene campos para ingresar el nombre y la descripción de la categoría.
  • Botones:
    • Guardar: Envía los datos del formulario para crear la categoría.
    • Cancelar: Redirige al usuario a la lista de categorías.
  • JavaScript: Maneja el envío del formulario de manera asíncrona utilizando fetch para enviar los datos a la ruta /categoria/guardar.

2. editar.html

Descripción: Este archivo permite al usuario editar una categoría existente.

Componentes:

  • Formulario: Incluye campos prellenados con el nombre y la descripción de la categoría a editar, además de un campo oculto para el ID de la categoría.
  • Botones:
    • Guardar Cambios: Envía los datos actualizados de la categoría.
    • Cancelar: Redirige al usuario a la lista de categorías.
  • JavaScript: Similar a crear.html, maneja el envío del formulario de manera asíncrona.

3. grilla.html

Descripción: Este archivo define un fragmento que se utiliza para mostrar una tabla de categorías.

Componentes:

  • Condicionales: Muestra un mensaje si no hay categorías disponibles.
  • Tabla: Lista las categorías con su ID, nombre y descripción.
  • Acciones: Incluye botones para editar y eliminar cada categoría, con confirmación para la eliminación.

4. listar.html

Descripción: Este archivo muestra una lista de todas las categorías registradas.

Componentes:

  • Formulario de Búsqueda: Permite al usuario buscar categorías por nombre.
  • Tabla: Presenta las categorías utilizando el fragmento de grilla.html.
  • Botón: Proporciona un enlace para crear una nueva categoría.

JavaScript: Maneja la búsqueda de categorías de manera asíncrona, actualizando la tabla sin recargar la página.

Conclusión

Estos archivos HTML forman parte de una interfaz de usuario para la gestión de categorías, permitiendo crear, editar, listar y eliminar categorías de manera eficiente. Cada archivo está diseñado para interactuar con el servidor a través de solicitudes asíncronas, mejorando la experiencia del usuario.
